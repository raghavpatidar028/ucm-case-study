02  —  Challenges & Response

What UAIR Is Dealing With  ·  What I Would Do

Based on UCM's public data profile and the UAIR mandate, these are the highest-leverage problems — and my direct response to each.

UAIR's Challenge
01
Banner + Workday Data Silos

Delayed cross-system reporting, inconsistent student records, manual reconciliation consuming UAIR analyst time.

02
Manual Ad-Hoc Reporting Burden

UAIR staff spend disproportionate time on repetitive data requests instead of strategic analysis.

03
No Predictive Retention Capability

At-risk students identified after withdrawal, not before. A 19-point gap between 4-yr and 6-yr graduation rates signals clear intervention opportunity.

04
Inconsistent Metrics & Governance

Departments make decisions on conflicting numbers. No shared data dictionary, no FERPA-aligned access controls.

My Response
01
Automated ETL — Python + SQL

Incremental load pipelines from Banner and Workday into a unified staging schema with validation checks and cross-system key integrity verification at each stage.

02
Power BI Automation + Parameterized Templates

Scheduled refresh pipelines and self-service report templates for recurring requests — targeting 30%+ reduction in ad-hoc burden within 6 months.

03
XGBoost Retention Model → Power BI

Logistic regression baseline, then XGBoost model using GPA, course engagement, enrollment history. Tiered alert output (Green/Yellow/Red) fed directly into advisor dashboards.

04
Data Dictionary + Row-Level Security

UCM Data Dictionary aligned to IPEDS definitions. Power BI RLS so advisors see only their advisees. Standardized DAX measure library across all dashboards.
